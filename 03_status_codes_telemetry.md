## ГЛАВА 1: Инфраструктурный фундамент (Сеть и Транспорт)
##  Чит №3: Коды ответов HTTP (Сигнальная телеметрия сервера)
## 1. Анатомия системы (Суть процесса)
Сервер ленив и не будет писать тебе длинных объяснений. В стартовую строку ответа он зашивает трехзначное число — HTTP Status Code. Это международный цифровой ГОСТ, по которому клиентский код мгновенно понимает физику результата обработки данных, даже не читая тело ответа.
Вся телеметрия разбита на 5 жестких классов по первой цифре:

* 1xx (Информационные): Процесс идет, труба открыта, жди. (На проде почти не встречаются).
* 2xx (Успех / Стыковка): Запрос принят, система отработала штатно, база обновилась.
* 3xx (Перенаправление / Редирект): Ресурс переехал. Иди по другому адресу.
* 4xx (Ошибка Клиента / Кривой бланк): Ты прислал мусор. Сервер защитился и заблокировал запрос. Проблема на твоей стороне.
* 5xx (Ошибка Сервера / Авария на складе): Бланк идеален, но у сервера взорвался внутренний движок, упала база данных или кончилась RAM. Проблема на стороне сервера.

Запомнить навсегда:
2xx — Прораб доволен, кирпич привезли.
3xx — Склад закрыт, разгружайся на соседнем участке.
4xx — Сам дурак, привез смету без подписи, кладовщик тебя выгнал.
5xx — Склад горит, кладовщик пьян, инженеры в панике.

##  2. Логическая цепь: 7 главных боевых кодов Системного Аналитика
Ты обязан знать физику этих семи кодов на уровне рефлексов:

   1. 200 OK (Штатный режим): Данные успешно прочитаны или изменены. Тело ответа содержит то, что ты просил.
   2. 201 Created (Объект создан): Чистый успех для POST-запросов. База данных приняла новую строку, сгенерирован новый ID.
   3. 400 Bad Request (Синтаксический брак): Клиент прислал битый JSON, забыл закрыть скобку или нарушил типы данных (передал текст вместо числа). Сервер не смог это распарсить.
   4. 401 Unauthorized (Нет паспорта): Ты забыл прикрепить токен/cookie в заголовки. Сервер тебя не знает.
   5. 403 Forbidden (Нет допуска): Сервер тебя знает, паспорт проверил, но у твоей роли (например, «Прораб») нет прав нажимать кнопку «Оплатить». Доступ заблокирован ролевой моделью.
   6. 404 Not Found (Пустота): Ошибка координат. Ты постучался в несуществующий путь (URL) или запросил ID строки, которой нет в базе данных.
   7. 500 Internal Server Error (Крах движка): Код сервера встретил необработанное исключение (капкан) и упал. Самая грязная ошибка. Ты сломал сервер своим запросом.

##  3. Канализация системных ошибок и капканов (Граничные условия)

* Капкан №1: Маскировка 200 OK (Ложь в теле ответа).
Самый частый косяк неопытных программистов. Сервер падает с внутренней ошибкой базы данных, но возвращает код 200 OK, а в теле ответа пишет текст: {"error": "База упала"}.
* Чем это грозит: Мобильное приложение или фронтенд смотрят только на код 200 OK в стартовой строке, думают, что всё супер, и пытаются отобразить пустые данные. Интерфейс намертво зависает или вылетает.
   * Решение Аналитика: Системный аналитик при проектировании ТЗ жестко прописывает: «В случае коллизии в БД система ОБЯЗАНА возвращать класс 4хх или 5хх, хардкод кода 200 на ошибки запрещен».
* Капкан №2: Разница между 401 и 403 (Ловушка путаницы).
На собеседованиях джуны постоянно путают эти коды.
* 401 — это отсутствие аутентификации (Кто ты? Повесь пропуск на грудь).
   * 403 — это отсутствие авторизации/прав (Я знаю, кто ты, Вася, но тебе в этот отсек нельзя).
* Капкан №3: Неуловимый 502 Bad Gateway.
Возникает, когда твой запрос долетает до внешнего распределителя (Nginx), но сам внутренний сервер с кодом (например, на Python) умер и не отвечает. Nginx понимает, что мост оборван, и выдает 502. Аналитик сразу понимает: сеть жива, упало само приложение.

##  4. Срез для собеседования в BigTech (Что спросит Лид)

* Вопрос: Каким методом (POST, GET, PUT) мы отправили запрос, если сервер вернул код 304 Not Modified, и что это значит?
* Ответ бьющий в ядро: Это сработал чит-код кеширования, чаще всего для метода GET. Это значит, что клиент запросил данные (например, тяжелый справочник ГЭСН), которые у него уже сохранены на ПК. Сервер проверил метку времени и ответил: «Данные на сервере не менялись с твоего последнего визита. Забирай их из своего локального кеша, тело ответа я не присылаю, экономим трафик трубы». Это экономит терабайты сети BigTech.
